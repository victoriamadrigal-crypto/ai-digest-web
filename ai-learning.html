El arte del prompt: cómo hablarle a Claude
⏱ 8 min · Fundamentos
✓
⌄

Por qué los prompts mediocres dan resultados mediocres

Claude es excepcionalmente capaz, pero necesita contexto para dar lo mejor de sí. La diferencia entre un prompt vago y uno bien construido puede ser la diferencia entre una respuesta genérica y un resultado que te ahorra horas de trabajo.

⚡ Regla de oro

Un buen prompt responde a cuatro preguntas: ¿Quién soy yo? (tu rol o contexto), ¿Qué necesito? (tarea), ¿Para qué? (objetivo), ¿En qué formato? (salida esperada).

Los 5 elementos de un prompt efectivo

1. Rol: Asigna un rol a Claude ("Actúa como diseñador UX senior")

2. Contexto: Da el contexto mínimo necesario para que entienda tu situación

3. Tarea: Sé específico sobre lo que pides, no lo que crees que quieres

4. Restricciones: Dile qué NO hacer ("sin tecnicismos", "máximo 3 opciones")

5. Formato: Define cómo quieres la respuesta (lista, tabla, párrafos, código)

Comparación práctica

❌ Prompt vago
Dame ideas para mejorar nuestra app.
✅ Prompt estructurado
Actúa como consultor de producto con experiencia en apps B2B SaaS.
Nuestra app de gestión de proyectos tiene un 40% de abandono en el onboarding.
Los usuarios llegan pero no completan la configuración inicial.
Dame 5 ideas concretas y priorizadas para reducir ese abandono, 
enfocándote en cambios de UX que no requieran rediseño completo.
Formato: lista numerada con título + 2 líneas de explicación cada una.

2
Contexto largo: Claude como tu memoria de trabajo
⏱ 6 min · Productividad
✓
⌄

La ventaja del contexto largo

Claude puede mantener conversaciones muy largas sin perder el hilo. Esto lo hace ideal para proyectos complejos donde necesitas que "recuerde" mucha información.

💡 Truco Pro

Al inicio de una conversación larga, dumpea todo el contexto relevante: documentos, decisiones previas, restricciones. Claude lo usará durante toda la sesión sin que tengas que repetirlo.

Técnica: el briefing inicial

En lugar de ir pregunta a pregunta, empieza con un "briefing" completo. Así Claude entiende el panorama completo y sus respuestas serán más coherentes y útiles.

📋 Plantilla de briefing inicial
## Contexto del proyecto
[Descripción en 2-3 líneas de qué es y para qué]

## Yo soy
[Tu rol y lo que sabe que es relevante]

## Restricciones importantes
- [Restricción 1]
- [Restricción 2]

## Lo que necesito en esta sesión
[Lista de tareas o preguntas que vas a trabajar]

¿Lo has entendido todo? Empieza con la primera tarea cuando estés listo.

Casos de uso perfectos para Claude

Análisis de documentos largos: Pega un contrato, informe o propuesta y hazle preguntas. Claude lo procesa todo.

Iteración de textos: Claude recuerda las versiones anteriores y entiende por qué quieres cambios.

Toma de decisiones compleja: Dale todos los pros y contras y pídele que razone contigo paso a paso.

✍️ Ejercicio — Práctica

Escribe tu propio briefing inicial

Piensa en un proyecto real en el que estés trabajando ahora. Escribe el briefing inicial que le darías a Claude usando la plantilla anterior.

## Contexto del proyecto
Estamos lanzando una nueva línea de producto para pymes en Q3.
Es un software de facturación. El mayor problema es que nuestros 
potenciales clientes usan Excel y no ven el valor de cambiar.

## Yo soy

No tengo equipo de diseño propio.

## Restricciones importantes
- No podemos prometer más de lo que el producto hace hoy
- El tono debe ser accesible, no tech-heavy
- Lanzamiento fijo: 15 de septiembre

## Lo que necesito en esta sesión
1. Propuesta de mensajes clave para la landing page
2. Ideas para campaña de email a leads existentes

3
Pensamiento paso a paso: razonamiento complejo
⏱ 7 min · Avanzado
✓
⌄

Chain of Thought: pídele que piense en voz alta

Para problemas complejos, pedir a Claude que "piense paso a paso" o que "muestre su razonamiento" mejora significativamente la calidad de sus respuestas. No es magia: simplemente le das espacio para no ir directo a la conclusión.

🔑 Frases clave

Añade al final de tu prompt: "Razona paso a paso" · "Muéstrame tu proceso de pensamiento" · "Antes de responder, explica tu enfoque" · "Piensa en voz alta"

Cuándo usar razonamiento paso a paso

✅ Úsalo cuando: Hay decisiones con trade-offs, análisis de datos complejos, problemas con múltiples variables, o cuando necesitas entender el "por qué" de la respuesta.

❌ No lo necesitas cuando: Pides escritura creativa, resúmenes simples, traducciones, o tareas de un solo paso.

🔍 Ejemplo: decisión de negocio
Tengo que decidir si contratar a un freelance por 3 meses (4.500€)
o invertir en una herramienta SaaS anual (2.400€/año).
La tarea es gestionar nuestras redes sociales.
Tenemos 2 personas en el equipo con algo de tiempo disponible.

Razona paso a paso considerando:
1. Coste real a 12 meses
2. Calidad y consistencia del output
3. Flexibilidad y riesgo
Y dame una recomendación final clara con su justificación.

Claude Code: programa sin saber programar
⏱ 10 min · Técnico
✓
⌄

¿Qué es Claude Code?

Claude Code es una herramienta de terminal que permite a Claude trabajar directamente en tu código: leer archivos, hacer cambios, ejecutar tests y solucionar bugs. No es solo un chatbot de código — es un agente que actúa en tu repositorio.

👥 Para no técnicos

Incluso si no sabes programar, puedes usar Claude Code para automatizar tareas repetitivas, generar scripts simples o entender código que te han mandado. No necesitas entender cada línea — necesitas saber qué resultado quieres.

Comandos básicos para empezar

💻 Flujo básico en Claude Code
# Instalar
npm install -g @anthropic-ai/claude-code

# Abrir en tu carpeta de proyecto
cd mi-proyecto
claude

# Ejemplos de lo que puedes pedirle:
> "Explícame qué hace este proyecto"
> "Encuentra y corrige el bug que hace que el login falle"
> "Añade validación de email al formulario de registro"
> "Escribe tests para la función de pago"
> "Refactoriza este archivo para que sea más legible"

El truco más potente: describir el resultado, no el código

No digas: "Crea una función async que haga fetch a la API y gestione errores". Di: "Cuando el usuario haga clic en Guardar, quiero que los datos se envíen al servidor y si hay un error, aparezca un mensaje en rojo debajo del formulario". Claude traduce el resultado al código.

✍️ Ejercicio — Práctica

Formula tu primera tarea para Claude Code

Piensa en algo que harías en tu trabajo que implique datos, archivos o repetición. Descríbelo como si se lo explicaras a un programador, enfocándote en el resultado que quieres.

✅ Ejemplo bien descrito para Claude Code
Tengo un CSV de ventas del año pasado con las columnas:
fecha, cliente, producto, cantidad, precio_unitario.

Necesito:
1. Calcular el total (cantidad × precio) para cada fila
2. Agrupar por cliente y sumar sus totales anuales
3. Ordenar de mayor a menor gasto
4. Exportar el resultado a un Excel nuevo llamado "top_clientes_2024.xlsx"

El archivo original se llama "ventas_2024.csv" y está en el escritorio.

Gemini ve lo que tú ves

Gemini puede analizar imágenes, gráficas, capturas de pantalla y PDFs directamente. Esto abre un mundo de posibilidades para tareas que antes requerían horas de trabajo manual.

📎 Cómo usarlo

En gemini.google.com, haz clic en el clip 📎 y sube una imagen, captura o PDF. Luego pregúntale sobre el contenido.

Qué puedes hacer con imágenes y PDFs

Gráficas e informes: Sube una gráfica y pide interpretación: "¿Qué tendencia muestra este gráfico y qué acciones sugeriría?"

Capturas de pantalla: Foto a un error en pantalla: "¿Qué está pasando aquí y cómo lo soluciono?"

Facturas y documentos: "Extrae todos los datos de esta factura en formato JSON"

Comparar diseños: "Compara estas dos versiones de la landing y dime cuál es más clara y por qué"

🖼️ Prompts para análisis visual
"Describe qué muestra esta imagen en detalle"
"¿Qué problemas de UX ves en esta captura?"
"Extrae todos los datos numéricos de este gráfico"
"¿Este diseño cumple con las buenas prácticas de accesibilidad?"
"Resume los puntos clave de este PDF en 5 bullets"

Gemini Deep Research

Deep Research es una función de Gemini Advanced que hace investigaciones exhaustivas de forma autónoma. No solo busca — razona, sintetiza y produce informes estructurados citando fuentes.

⏱ Cuándo usarlo

Cuando necesites un análisis que normalmente te llevaría 2-4 horas de investigación manual: análisis de mercado, benchmark de competidores, investigación de tendencias, due diligence inicial.

Cómo activarlo

🔍 Ejemplos de Deep Research
"Haz un análisis del mercado de herramientas de gestión de proyectos 
para equipos de 10-50 personas en España. Incluye los 5 principales 
players, sus precios, sus puntos fuertes/débiles y las tendencias

⏱ 8 min · Personalización

⌄

¿Qué son los Gems?

Los Gems son versiones personalizadas de Gemini con instrucciones específicas, tono y contexto fijo. Creas un Gem una vez y lo reutilizas siempre que quieras, sin tener que repetir el contexto.

💎 Dónde crearlos

En gemini.google.com → "Mis Gems" en el panel izquierdo → "Nuevo Gem". Define el nombre, instrucciones y comportamiento.

Ideas de Gems para tu equipo

Gem "Revisor de Emails": Revisa mis emails y mejora el tono, claridad y profesionalidad. Soy director de operaciones de startup tech.

Gem "Analista de Datos": Analiza datos que te comparta e identifica siempre: tendencias, anomalías y acciones recomendadas. Somos una empresa de retail online.

Gem "Facilitador de Reuniones": Toma mis notas caóticas de reunión y devuélvelas como: resumen ejecutivo + decisiones tomadas + próximos pasos con responsables.

💎 Plantilla para crear un Gem
Nombre: [Nombre descriptivo] Instrucciones: Eres un asistente especializado en [área]. Tu usuario es [rol] en [tipo de empresa/sector]. Siempre que te contacten: - [Comportamiento 1] - [Comportamiento 2] - [Comportamiento 3] Tono: [formal/casual/técnico/accesible] Formato de respuesta preferido: [bullets/párrafos/tablas] Nunca: - [Restricción 1] - [Restricción 2]
✍️ Ejercicio — Práctica

Diseña tu primer Gem

Piensa en la tarea repetitiva de tu trabajo donde más tiempo pierdes explicando contexto. Diseña un Gem que lo resuelva usando la plantilla anterior.

✅ Ejemplo de Gem bien definido
Nombre: Redactor de Propuestas Comerciales

Instrucciones:
Eres un experto en redacción de propuestas comerciales B2B.
Tu usuario es del equipo de ventas de una empresa de consultoría IT.

Cuando te compartan información de un cliente o proyecto:
- Estructura siempre la propuesta en: Resumen ejecutivo → Problema identificado → Solución propuesta → Por qué nosotros → Inversión → Próximos pasos
- Usa lenguaje orientado a beneficios, no a características
- Adapta el tono al sector del cliente (más formal para banca, más directo para startups)

Tono: profesional pero cercano
Formato: estructura con títulos, párrafos cortos y bullets donde aplique

Nunca:
- Inventes datos o estadísticas
- Uses jerga demasiado técnica sin explicarla

Kiro

El IDE agéntico de AWS que va más allá de generar código: planifica, especifica y construye features completas desde tu descripción en lenguaje natural.

1
Kiro vs otros IDEs IA: qué lo hace diferente
⏱ 8 min · Fundamentos
✓
⌄

El problema del "vibe coding"

La mayoría de herramientas de IA para código hacen "vibe coding": tú describes algo vagamente, la IA genera código, tú corriges, repites hasta que funciona. Para proyectos pequeños está bien. Para proyectos reales, es caótico.

👻 Dato curioso

El logo de Kiro es un fantasma. Su nombre significa "encrucijada" en japonés (きろ). Es un IDE basado en VS Code desarrollado por AWS.

La diferencia: Spec-Driven Development

Kiro introduce un flujo diferente. Antes de escribir una línea de código:

1. Requirements: Convierte tu descripción en criterios de aceptación formales

2. Design: Propone la arquitectura y decisiones técnicas

3. Tasks: Genera un plan de implementación con tareas ordenadas por dependencia

Solo entonces empieza a escribir código.

⚡ Kiro vs Cursor — en una frase
Cursor: "genera código rápido, itera"
Kiro:   "planifica primero, implementa con estructura"

¿Cuándo usar cada uno?
→ Bug fix rápido o prototipo: Cursor o Claude Code
→ Feature nueva en proyecto grande: Kiro

2
Tu primera Spec: de idea a plan de implementación
⏱ 10 min · Práctico
✓
⌄

Cómo crear una Spec en Kiro

En Kiro, abre el panel de chat → selecciona "Spec" → describe tu feature. Kiro hará preguntas de clarificación y luego generará tres documentos: requirements.md, design.md y tasks.md.

📋 Lo que genera Kiro

requirements.md: Historias de usuario + criterios de aceptación · design.md: Arquitectura + decisiones técnicas · tasks.md: Lista de tareas ordenadas con dependencias

Cómo describir una feature a Kiro

No necesitas usar jerga técnica. Describe la feature desde la perspectiva del usuario final:

✅ Descripción efectiva para Kiro Spec
Quiero añadir un sistema de invitaciones a nuestra app.

Los usuarios deberían poder:
- Invitar a compañeros por email desde su cuenta
- Los invitados reciben un email con un link que les pre-rellena el registro
- El usuario que invita ve en su perfil cuántas invitaciones ha enviado y cuáles han aceptado
- Los admins pueden ver un listado global de todas las invitaciones

Restricciones:
- Cada usuario puede invitar máximo 10 personas
- Las invitaciones caducan a los 7 días
- No reinvitar a emails que ya son usuarios

Después de la Spec: el bucle de implementación

Una vez Kiro genera las tasks, puedes ir tarea a tarea activando el modo Autopilot o aprobando cada cambio manualmente. Para proyectos importantes, aprobación manual siempre.

✍️ Ejercicio — Práctica

Escribe tu primera descripción de Spec

Elige una feature que tu equipo necesite añadir a algún sistema o app. Descríbela como se la explicarías a Kiro, desde la perspectiva del usuario.

✅ Ejemplo de Spec bien escrita
Quiero añadir un dashboard de métricas para managers en nuestra 
herramienta de gestión de proyectos.

Los managers deberían poder:
- Ver un resumen semanal del avance de todos sus proyectos activos
- Identificar rápidamente qué proyectos están en riesgo (atrasados o bloqueados)
- Ver la carga de trabajo de cada miembro del equipo (tareas asignadas vs completadas)
- Exportar el resumen en PDF para enviarlo a dirección

Restricciones:
- Solo accesible para roles "Manager" o "Admin", no para empleados base
- El dashboard debe cargar en menos de 2 segundos
- Los datos deben estar actualizados en tiempo real
- En móvil debe verse correctamente (responsive)
- Sin gráficas complejas: solo números y colores (verde/amarillo/rojo)

Agent Hooks: automatiza tu flujo de desarrollo
⏱ 8 min · Avanzado · Técnico
✓
⌄

¿Qué son los Agent Hooks?

Los Hooks son automatizaciones que se ejecutan cuando pasan cosas en tu proyecto: cuando guardas un archivo, cuando creas uno nuevo, cuando haces un commit. Son como GitHub Actions pero dentro de tu IDE y con IA.

⚡ Para no técnicos

Imagina que cada vez que guardas un archivo de código, automáticamente se actualiza la documentación. O que cuando añades una función, se generan los tests automáticamente. Eso es un Hook.

Hooks más útiles para equipos

Auto-documentación: "Cuando guarde cualquier función nueva, genera automáticamente el comentario JSDoc explicando qué hace, qué recibe y qué devuelve"

Tests automáticos: "Cuando añada una función nueva a utils/, genera un test unitario básico en el archivo de tests correspondiente"

Revisión de seguridad: "Cuando modifique archivos de autenticación, revisa que no haya vulnerabilidades obvias y añade un comentario con el resultado"

🔧 Cómo definir un Hook en Kiro
# En Kiro: panel lateral → "Hooks" → "New Hook" Trigger: On file save Filter: *.py (solo archivos Python) Prompt del agente: "Cuando se guarde este archivo: 1. Revisa si hay funciones sin docstring 2. Añade docstrings en formato Google Style donde falten 3. Si la función es compleja (más de 20 líneas), añade comentarios inline No modifiques la lógica del código, solo la documentación."

Steering files: guía el comportamiento global de Kiro

Los archivos de Steering (.kiro/steering/) son instrucciones permanentes para Kiro sobre cómo debe comportarse en tu proyecto. Define convenciones de código, arquitectura preferida, qué librerías usar o prohibir.
